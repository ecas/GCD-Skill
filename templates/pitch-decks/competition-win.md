# Audience: CTO, CIO, IT Director, Procurement — evaluating or leaving a competitor
# Length: 12 slides | Target duration: 45–60 minutes
# Use when: Active RFP evaluation, contract renewal with competitor, explicit "we're
#            evaluating AWS/Azure/Microsoft" conversation

## Slide 8 — Migration Path

**Headline:** Switching Is Not a Big-Bang Risk — Here's the Path

**Migration from {Competitor} to GCP:**

**Common concerns addressed:**
| Concern | Answer |
|---------|--------|
| "We have everything in {competitor} — migration is too risky" | Phased migration: keep {competitor} for low-priority workloads during transition. Run in parallel. |
| "Our team only knows {competitor} tools" | Google provides training, certifications, and a dedicated CE during migration |
| "We've signed a {N}-year commitment with {competitor}" | We can work around contract timing — start with new workloads, migrate the rest at renewal |
| "Data egress costs will make migration expensive" | Google offers migration credits. We'll model the egress cost in the TCO analysis. |

**Phased migration approach:**
- Phase 0 (Assessment): Free — Stratozone / CAST discovery tool
- Phase 1 (New workloads): New applications deployed on GCP, existing on {competitor}
- Phase 2 (Migrate): Move workloads in waves, prioritized by risk/value
- Phase 3 (Optimize): Decommission {competitor} dependency, realize full savings

**Partner support:** {Partner name} has certified migration engineers for {competitor} → GCP transitions

**Speaker notes:**
> Migration risk is the #1 objection in displacement deals. The phased approach answers it directly. Emphasize: "You don't have to decide on a full migration today. Start with one new workload, validate the experience, then scale." Low-risk entry reduces the perceived commitment required to say yes.

## Usage Notes

- This deck is most effective when the customer's evaluation criteria (slide 2) are filled in during the session, not pre-populated by Google.
- Slides 5–7 (differentiators) should be selected and customized based on the criteria that emerge in slide 2. Do not present all three differentiator options — pick the most relevant.
- Competitive claims must be factual and defensible. Do not include claims you cannot back with a specific reference, benchmark, or architecture point.
- For AWS displacement: emphasize simplicity, AI/ML leadership, TCO on data-intensive workloads, network performance.
- For Microsoft displacement: emphasize AI/ML independence (not dependent on OpenAI licensing), open platform, BigQuery vs. Synapse, pricing transparency.
- For on-premises displacement: emphasize total cost, elasticity, talent, speed of innovation.
- For Polish-language delivery: translate all narrative content; product and company names remain in English.
